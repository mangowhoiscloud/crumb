# Reba Berserker — spec.md

## Title
**Reba Berserker** — a 60-second portrait-mobile arena hommage to 레바의 모험 (2007-era Korean Flash adventure-action) with a Berserker rage-burst mode as its core satisfaction loop.

## Goal
Build a self-contained Phaser 3.80+ multi-file PWA under `artifacts/game/`. The player taps to slash, swipes to dash, and racks up kills inside a 60-second timer. Every 10 kills auto-triggers an 8-second Berserker burst that triples score, widens slash radius 1.5×, applies a red screen tint and hit-stop. Win = score ≥ 50 within the timer. Lose = HP reaches 0.

## Acceptance Criteria

> All ACs are externally testable from a running build. AC1 is compiled to a deterministic browser-side predicate (see `data.ac_predicates`). AC2–AC7 are graded by the verifier through a Playwright MCP playthrough scenario.

- **AC1**: On load, a single `<canvas>` element is rendered in portrait orientation with width between 320 and 428 px and height greater than width (`Phaser.Scale.FIT` portrait), inside a viewport meta tag with `touch-action: none`.
- **AC2**: A pointerdown event on the canvas produces a visible slash sprite within 80ms, oriented from player position toward the nearest enemy (or toward the tap point if no enemy is on-screen). Tapping with no enemies still shows a slash to confirm input responsiveness.
- **AC3**: After 10 kills the rage bar fills to 100% AND the screen tints red (rgba(214,40,40,0.30) overlay) AND a `Berserker!` HUD callout appears — all three within the same frame as the 10th kill being registered.
- **AC4**: Berserker mode lasts 8000 ± 200 ms, then the red tint fades out over 400ms, the rage bar resets to 0, and the score multiplier returns to its normal combo-only value.
- **AC5**: Enemy contact with the player decrements one of three visible HP heart icons in the top-right HUD. Contact during the 200ms after a contact registers is ignored (i-frames) so a single brush doesn't drain multiple hearts.
- **AC6**: When the 60-second timer reaches 0, the timer halts at `0:00`, all enemy spawns stop, the player can no longer slash or dash, and a final-score readout is shown on screen along with a `RESTART` button.
- **AC7**: When HP reaches 0, the run ends immediately (within 100ms): timer halts, enemies stop, the canvas dims, and a `RESTART` button is shown. Tapping `RESTART` resets score, HP, rage, timer, and re-enters the run.

## Rule Book (binding tuning — see `tuning.json` for numeric authority)

- **Grid / arena**: top-down portrait arena. Player anchored near vertical center; enemies spawn from screen edges and walk toward player.
- **Player input**: single-tap = slash toward nearest enemy (or toward tap point if none); swipe (>30px in <300ms) = dash 80px in swipe direction with 200ms invulnerability.
- **Slash**: 60px base radius, 1.5× during berserker, 120ms animation, 50ms hit-stop on each kill, 80ms × 4px screen-shake (2px under berserker — anti-nausea cap).
- **Score**: +1 per kill, ×combo_multiplier, ×3 berserker multiplier when active. Combo multiplier grows 1.0 → 1.2 → 1.5 → 2.0 → 5.0 within an 800ms chain window; resets when window expires.
- **Rage**: +10 per kill (cap 100). At 100, Berserker auto-activates: 8000 ± 200 ms duration, +200ms tint fade-in, +400ms tint fade-out, rage resets to 0 on deactivation. Cannot manually trigger or extend.
- **HP**: 3 hearts. Enemy contact = -1 heart, then 200ms i-frames. HP 0 ends the run.
- **Spawn cadence**: linear ramp `1500ms → 600ms` over 60s. First 5s is the "free win" warm-up — single enemy, slow spawn. After 5s, swarm density carries the difficulty.
- **Win check**: at t=60s, score ≥ 50 = WIN screen; otherwise = LOSS screen. HP=0 anywhere is immediate LOSS.

## Constraints

- **Stack**: Phaser 3.80+ via CDN, multi-file PWA per `agents/specialists/game-design.md` §1.1. No build step. ES modules + import map.
- **Asset pipeline**: procedural sprite generation in `BootScene.js` (Canvas → texture). No binary asset files unless an atlas is genuinely needed (≤25 files total, ≤5MB).
- **Audio**: Web Audio synth in `systems/AudioManager.js` — 1 BGM lead + 5 SFX (slash, hit, kill, berserker-trigger, game-over). No external audio files.
- **Performance**: 60fps on iPhone Safari 16+ / Android Chrome 100+ at peak swarm.
- **Mobile-first**: viewport 320–428 portrait, `touch-action: none`, ≥44×44 hit zones for menu buttons, safe-area-inset-top/bottom 24px.
- **Persistence**: NONE — no Postgres / Supabase / leaderboard. Score is in-memory only.
- **Network**: zero runtime requests after CDN load + first SW fetch. Game runs offline.
- **PWA**: `manifest.webmanifest` + `sw.js` cache-first; icon-192/512 emitted as base64 PNG generated by Canvas in BootScene.
- **Color contrast**: every text + accent token ≥ 4.5:1 contrast on its background (`#D62828` rage red on `#3D2817` wood-brown = 4.8:1 ✓).
- **Reduced motion**: `prefers-reduced-motion: reduce` halves shake amplitude, replaces flicker tint with steady tint.

## Non-Goals

- ❌ Multiplayer / leaderboards / score persistence (P0 scope is solo offline)
- ❌ IAP / ads / monetization surfaces
- ❌ Multiple weapons or character upgrades — single slash, single dash
- ❌ Multiple stages or levels — one 60s arena
- ❌ Tutorial overlay — 5s "free win" warm-up serves as implicit tutorial
- ❌ Localization beyond Korean+English mixed UI strings (`SCORE`, `RAGE`, `RESTART`)
- ❌ Real-time physics — arcade physics + tween-driven enemy walk only
- ❌ Sprite asset files — procedural Canvas-rendered sprites only
- ❌ Frame-perfect speedrun timing — 800ms combo window is forgiving on purpose
