## 4. Crumb's Position — Core Framing Message

### Synthesis of Facts

```
Bagelcode production stack       = Unity
Similar genre leader (Royal Match) = Unity
New-game team hire spec            = Unity
Korean competitor (Pixie) trajectory = AI workflow + solo / 3-week launch
LLM-friendly prototype framework   = Phaser HTML5 (94% first-attempt)
LLM-generated mobile binary        = Blocked by App Store policy (Lovable response)
```

### Precise Meaning of Our Decision

> ❌ "**Compete with Unity**" — Crumb as a prototype builder replacing Unity.
>
> ✅ "**Pre-stage tool for Unity**" — Validate the concept within 30 minutes → only validated concepts go to the Unity team for productionization.

Why this framing is strong:
- Preserves and reinforces Bagelcode's existing stack (Unity)
- Doesn't intrude on the new-game team's production infrastructure
- Targets LLM's strength (fast HTML5 prototyping) directly
- Consistent with the DoubleU Games trajectory (accelerated AI workflow)
- Makes the **stage** of "planner has the agent build a game" clear

### Crumb's Lifecycle Position

```
[Planner's one-line idea]
        │
        ▼
  ┌──────────────────┐
  │  Crumb            │  ← 30 min, HTML5 + Phaser, instant browser play
  │  (LLM agents)     │     validate / reject
  └────────┬─────────┘
           │  (validated spec + DESIGN.md + tuning)
           ▼
  ┌──────────────────┐
  │  Unity new-game   │  ← days~weeks, native iOS/Android
  │  (humans + Unity AI)│   production build
  └────────┬─────────┘
           │
           ▼
   App Store / Google Play
```

→ **The two stacks are sequential, not competing.** Crumb produces the input for Unity.

### Outputs = Inputs to the Unity Team

The 4 things Crumb produces in its final turn:

| Output | Unity team receives as |
|---|---|
| `game.html` (Phaser demo) | Convert to concept demo video |
| `spec.md` (AC + rulebook) | Base for the Unity GDD |
| `DESIGN.md` (color / mechanics / motion) | Base for the Unity Designer |
| `tuning.json` (balance values) | Imported into Unity ScriptableObject |

→ The Phaser game.html itself is throwaway. **The 4 — concept · spec · DESIGN · tuning — are the input assets for the Unity team**. That's Crumb's real deliverable.
